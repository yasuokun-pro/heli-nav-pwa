# HELI NAV — 公開・インストール手順

このフォルダ一式(index.html / manifest.json / sw.js / icon-192.png / icon-512.png)を
HTTPSのサーバーに置くだけでPWAとして動きます。以下はGitHub Pagesを使う場合の手順です(無料)。

## 1. GitHubに公開する(初回のみ・約5分)

1. https://github.com でアカウント作成(無料)
2. 右上「+」→「New repository」
   - Repository name: `heli-nav`(任意)
   - Public を選択 → Create repository
3. 「uploading an existing file」をクリックし、このフォルダの5ファイルを
   まとめてドラッグ&ドロップ → Commit changes
4. リポジトリの Settings → Pages を開き、
   - Source: Deploy from a branch
   - Branch: `main` / `(root)` → Save
5. 1〜2分待つと `https://ユーザー名.github.io/heli-nav/` でアプリが公開されます

## 2. 端末にインストールする

- **iPhone/iPad (Safari)**: 上記URLを開く → 共有ボタン → 「ホーム画面に追加」
- **Android (Chrome)**: URLを開く → メニュー(⋮) → 「アプリをインストール」
  (または画面下に出るインストールバナー)

以降はホーム画面のアイコンから全画面で起動します。

## 3. オフラインで使うコツ

- アプリ本体は自動でキャッシュされ、圏外でも起動します
- 地図タイルは「一度表示した範囲」がキャッシュされます。
  **飛行前に、使うズームレベルで運航エリアをひと通りなぞって表示**しておいてください
- タイルキャッシュは約4,000枚で古いものから自動削除されます
- 雨雲・他機情報・新しい地図範囲の取得は通信が必要です

## 4. データについて

- ルート・設定値・飛行ログは端末内(localStorage)に自動保存されます
- ブラウザの「サイトデータ削除」を行うと消えるので、
  大事なログは定期的にCSV書き出しを推奨します

## 5. アプリを更新するとき

1. 新しい index.html 等をリポジトリにアップロード(同名で上書き)
2. `sw.js` の先頭にある `const VER = 'hnav-v5-1'` の数字を上げる(例: v5-2)
   → 各端末は次回起動時に自動で新バージョンに入れ替わります

## 注意

本アプリは開発中のプロトタイプです。表示される方位・燃料・空域・交通・気象は
参考値であり、正式な航法情報源としては使用できません。
空域は必ず最新のAIP・エンルートチャートで確認してください。
